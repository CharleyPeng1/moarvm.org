Native Calling Interface

Port the 6model/NQP native calling support to also run on MoarVM. This will enable the NativeCall module to run on MoarVM, in turn opening the door to many other modules.

Support Perl 6 Concurrency

MoarVM does already have basic thread support. Now it needs to be exposed for use by Rakudo Perl 6, and stress-tested. Mechanisms such as locks, semaphores, queues, and timers also need to be provided, in order that Rakudo's implementation of schedulers, promises, channels, and supplies can be provided. This includes providing asynchronous I/O.

Bytecode Specialization

The initial round of optimization that MoarVM will do is bytecode specialization: using runtime information to produce smarter, smaller bytecode, including using a number of internal operations that it can prove are safe.

NFG Support

Provide Unicode strings at grapheme level.

Inlining and Escape Analysis

Teach the specializer to inline and perform escape analysis, and do the allocation transformations it allows.

JIT Compilation

Turn specialized bytecode into machine code, on hot paths.
